⚡ ALGORITHM REFERENCE

Static Protocol BASELINE
algorithm :: deterministic
Always uses Tag=64, PA=0.5 (action 14) regardless of channel conditions. Represents standard industry QKD deployments. Wastes capacity in good conditions and breaches during turbulence or eavesdropping. Serves as the 1.0× baseline against which all other algorithms are compared.
▸ Performance: 1.0× yield  |  ~20% failure rate
Random Policy BASELINE
algorithm :: stochastic-baseline
Selects uniformly random actions from the 40 possible choices (4 tag lengths × 10 PA ratios). A sanity-check baseline — any trained agent should significantly outperform random. Expect many breaches and failures due to no awareness of channel state.
▸ Performance: highly variable  |  ~50%+ failure rate
Naive DQN (No Stabilization) FROM PAPER
algorithm :: reactive-RL
A basic Deep Q-Network that only observes current QBER without ΔQBER momentum. No target network stabilization (no Polyak averaging). Key weaknesses:
• No pre-emptive throttling before turbulence onset
• Noisy Q-value oscillations cause erratic decisions
• Overshoots PA ratio during rapid QBER transitions
This represents the "first-attempt" RL approach referenced in the comparative analysis table.
▸ Performance: 1.2× yield (unstable)  |  ~15% failure rate
Context-Aware DQN (Heuristic) FROM PAPER
algorithm :: rule-based-approximation
A JavaScript hand-coded approximation of the trained Context-Aware DQN's behavior. Uses ΔQBER for pre-emptive throttling and Devetak-Winter bounds for safe PA selection. Runs entirely in the browser without the Python backend.

Implements the same 7D state space awareness (QBER, ΔQBER, SNR, previous PA, tag length, key pool, episode progress) but via hand-crafted rules rather than learned weights.
▸ Performance: ~1.79× yield  |  ~8% failure rate (approx)
Context-Aware DQN (Trained Model) ★ OURS ★
algorithm :: neural-network (Dueling Double DQN + PER)
The actual Dueling Double DQN with Prioritized Experience Replay, trained over 5000 episodes in PyTorch. Uses the full 7D state space:
• QBER, ΔQBER (momentum), SNR, previous PA ratio
• Tag length, key pool level, episode progress

Architecture: Shared 256-unit feature layer → Value stream (256→128→1) + Advantage stream (256→128→40). Polyak soft-target updates (τ=0.005), cosine annealing LR schedule, β-annealed priority sampling.

Runs real-time inference via the Python backend (port 8001). This is our proposed model from the paper.
▸ Performance: 1.79× yield  |  ~8% failure rate  |  55.89% fewer aborts
